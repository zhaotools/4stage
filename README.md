# Weinstein 四阶段趋势分析 V1.0.13

GitHub Pages 前端 + GitHub Actions 后台数据服务。浏览器不再直连第三方行情接口，也不在本地执行阶段分析。

## 部署

1. 在 GitHub 新建仓库，例如 `4stage`。
2. 把本目录中的全部文件上传到仓库根目录。
3. 打开仓库 **Settings → Pages**。
4. 在 **Build and deployment** 中选择 **Deploy from a branch**。
5. Branch 选择 `main`，目录选择 `/ (root)`，点击 **Save**。
6. 等待约 1–3 分钟，通过 GitHub 提供的网址访问。

如果仓库名是 `4stage`，地址通常为：

`https://你的用户名.github.io/4stage/`

## 资产范围

- 加密蓝筹及相关股票：BTC、ETH、HYPE、MSTR、CRCL、HOOD、COIN
- A股：常用 ETF、主要指数、沪深300成分股
- 美股：主流高成交量个股与常用 ETF
- 页面提供“加密蓝筹”“A股ETF”“美股ETF”三个快捷下拉框，全部资产也可通过代码或名称搜索。

## 数据说明

- GitHub Actions 每日运行 `backend/update.mjs`。
- 后台从 Binance 获取周线，并从东方财富、Yahoo Finance 获取日线后聚合周线，统一完成四阶段分析并生成 JSON。
- 阶段判断只使用已经收盘的完整周线；未完成的本周行情不会参与确认。
- K线、最新价格和最新周涨跌显示数据源提供的最新行情；页面分别注明“K线更新至”和“阶段确认至”日期。
- 价格显著跌破或站回30周均线时，优先解决 S2/S4 硬冲突，并显示“破位转换”或“修复转换”状态。
- 阶段转换采用状态机持续跟踪，连续两根完整周线满足目标阶段条件后才确认；转换失效则恢复原确认阶段。
- 页面直接展示支持当前阶段的核心证据，包括价格与MA30、MA30方向、10/30周结构和52周位置，并标出支持、中性或分歧状态。
- 手机端三个快捷下拉框保持同一行，选择后显示资产代码；阶段证据移动到历史K线下方。
- PC端扩大历史周线的纵向绘图区，并让图表模块底部与左侧阶段证据卡片大致对齐。
- 每次发布前运行15个典型资产回归案例，以及未完成周线、转换确认、转换失效防护测试。
- GitHub Pages 浏览器只读取仓库网站下的 `data/*.json` 结果并负责显示。
- A股优先使用东方财富，失败后尝试 Yahoo Finance，两个在线接口均不可用时才回退到上次成功缓存。
- 免费接口可能存在延迟或短时不可用，不适合作为正式商业数据源。

## 文件

- `index.html`：页面结构
- `styles.css`：深色交易终端 UI
- `app.js`：读取后台结果和 K 线绘制
- `backend/update.mjs`：后台行情获取与结果生成
- `backend/stage-engine.mjs`：完整周线过滤、阶段评分、硬冲突与阶段转换引擎
- `tests/stage-regression.mjs`：典型资产和完整周线回归测试
- `config/universe.json`：资产清单和分类
- `.github/workflows/pages.yml`：每日后台更新与网站发布
- `.nojekyll`：避免 GitHub Pages 的 Jekyll 处理
